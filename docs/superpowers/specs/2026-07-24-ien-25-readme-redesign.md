# IEN-25 README 개편 설계

## 목표

`ienlab_react_microsite`의 `/about` 페이지에 있는 최신 정보를 기준으로 GitHub 프로필용 `readme.md`를 전면 개편한다. 방문자가 별도의 페이지를 열지 않아도 IENGROUND의 소개, 학력, 기술, 활동, 장비, 연락처를 모두 확인할 수 있어야 한다.

## 정보 원본

- 소개 문구와 화면 구성: `src/ui/public/about/AboutScreen.tsx`
- 한국어 소개 문구와 섹션 제목: `src/locales/ko/strings.json`
- 기존 상단 이미지와 저장소 링크: 현재 `readme.md`

원본의 한국어 내용을 우선하며 기술명, 제품명, 서비스명은 원문 표기를 유지한다.

## 문서 구조

1. 기존 상단 대표 이미지
2. 인사말과 자기소개
3. 학력 및 기술 스택
4. 주요 발자취
5. 대학 활동
6. 작업 장비
7. 연락처

## 표현 방식

- GitHub가 허용하는 HTML 태그와 속성을 적극 사용해 `/about`의 그리드와 카드 배치를 최대한 재현한다.
- 임의 CSS와 `<style>` 태그에는 의존하지 않고 `<table>`, `<tr>`, `<td>`, `<picture>`, `<img>`, `<p>`, `<h1>`~`<h3>`, `<a>`, `<code>`를 중심으로 구성한다.
- 모든 주요 섹션은 왼쪽에 두 자리 인덱스와 라벨, 오른쪽에 본문이 놓이는 2열 구조로 통일한다.
- 소개는 프로필 이미지와 본문을 2열로, 기술 스택은 3열 카드로, 주요 발자취는 4열 하이라이트와 전체 타임라인으로, 대학 활동은 3열 하이라이트와 전체 타임라인으로 배치한다.
- 작업 장비는 4개 분류를 열 단위로 배치하고 각 장비의 이미지, 이름, 설명을 함께 노출한다.
- 기술 스택은 개발, 디자인, 도구 및 작업 흐름의 세 그룹으로 구분한다.
- 장비는 주 개발 장비, 주 모바일 기기, 음향 장비, 기타 장비로 구분하고 `/about`의 설명을 모두 싣는다.
- 연락처는 포트폴리오, GitHub, Instagram, 이메일, 블로그를 명시적인 링크로 제공한다.
- 긴 정보를 숨기는 접이식 요소는 사용하지 않는다. 모든 내용을 기본 상태에서 노출한다.
- 프로필, 로고, 장비 10건은 실제 파일을 추가하지 않고 교체 가능한 외부 자리표시자 `<img src="...">`로 둔다.
- 각 이미지 바로 앞에 `src`만 교체하면 된다는 HTML 주석을 둔다.
- 기존 상단 대표 이미지는 유지한다.

## 이미지 자리표시자

- 프로필 이미지 1개
- 로고 이미지 1개
- 장비 이미지 10개

자리표시자는 `https://placehold.co/` 주소를 사용한다. 실제 이미지 적용 시 각 `<img>`의 `src` 값만 변경하며 너비, 대체 텍스트, 배치 마크업은 유지한다.

## 범위

### 포함

- `/about`에 정의된 소개 문구 전체
- 학력 2건
- 기술 스택 3개 그룹 전체
- 주요 발자취 7건
- 대학 활동 9건
- 작업 장비 10건과 설명 전체
- 연락처 5종

### 제외

- `/about` 화면의 애니메이션과 반응형 레이아웃 재현
- GitHub가 제거하는 사용자 정의 CSS와 웹사이트 전용 애니메이션
- 영어 번역문 병기
- 실제 프로필·로고·장비 이미지 파일 추가
- 현재 작업 트리에 존재하는 `.gitignore` 변경

## 검증 기준

- 원본 데이터의 항목 수와 README 항목 수를 대조한다.
- 날짜, 명칭, 역할, 수상 내역, 장비 설명에 누락이나 오기가 없는지 확인한다.
- 모든 외부 링크가 올바른 형식인지 검사한다.
- HTML 태그의 열림·닫힘 쌍과 중첩 구조를 검사한다.
- 교체 대상 이미지 자리표시자 12개와 `src` 속성이 모두 존재하는지 확인한다.
- GitHub Markdown 렌더링 결과에서 주요 표, 이미지, 링크가 제거되지 않는지 확인한다.
- 최종 변경 범위가 `readme.md`와 이 설계·계획 문서로 제한되는지 확인한다.
